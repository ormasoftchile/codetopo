## Work Log

### Problem Identified
File nodes (node_type='file', file_id=NULL) were not cascade-deleted when `persist_file()` re-persisted the same path. The `DELETE FROM files WHERE path = ?` cascades to symbol nodes (file_id=N), but file nodes (file_id=NULL) survive due to NULL not participating in FK cascade.

### Solution Implemented
1. Added explicit `DELETE FROM nodes WHERE node_type='file' AND name = ?` immediately before file_node INSERT
2. Added `sqlite3_step` return value check to catch and report `SQLITE_CONSTRAINT` errors with diagnostics
3. Follows existing inline prepare/step/finalize pattern (prepared statement caching deferred to DEC-027)
