## Design Decisions to Lock In

**DEC-035: Parser Reuse Pattern**
- Parser objects created once per language, reused across files
- Thread-local storage (no locks needed)
- Fallback: Parse with fresh parser if creation fails

**DEC-036: Language Dispatch Strategy**
- Pre-computed language ID enum (C=0, CPP=1, CSHARP=2, ...)
- No string comparisons in hot loop
- All language-specific logic via switch statement

**DEC-037: File Read Buffering**
- Pre-allocate string to exact file size before read
- No exponential growth, no copy overhead
- Apply to all file sizes (512B → 512MB)
